# Decision Process

This document defines how governance and policy decisions are proposed, reviewed, and recorded.

## Decision Types

### Routine Decisions

Examples: label cleanup, issue triage, small documentation fixes, and straightforward first
contributions.

Decision owner: relevant maintainer.

### Maintainer Decisions

Examples: accepting membership applications, accepting project proposals, changing repository
permissions, and changing project roadmap direction.

Decision owner: organization or project maintainers.

### Governance Decisions

Examples: changing the Code of Conduct, changing access policy, changing project admission rules,
creating/removing teams, and archiving a project.

Decision owner: organization maintainers, with owner final decision if unresolved.

## RFC / Policy Change Flow

1. Open a governance proposal issue.
2. Explain problem, proposed change, alternatives, risks, and rollout plan.
3. Allow maintainer review.
4. Update docs in a PR if accepted.
5. Close the issue with the final decision.

## Decision Log

Important decisions should be recorded in `docs/DECISION_LOG.md`.

Record:

- date;
- decision;
- reason;
- affected repositories/docs;
- follow-up tasks.
