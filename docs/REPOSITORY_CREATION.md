# Repository Creation Policy

New repositories should be created only after a project proposal is accepted or an organization
maintainer approves a clear operational need.

## Creation Checklist

- [ ] Accepted project proposal or approved operational need exists.
- [ ] Repository name is clear and lowercase where practical.
- [ ] Visibility is chosen deliberately.
- [ ] License is selected.
- [ ] Maintainer team is assigned.
- [ ] Baseline files are added:
  - `README.md`
  - `LICENSE`
  - `CONTRIBUTING.md` or organization link
  - `CODE_OF_CONDUCT.md` or organization link
  - `SECURITY.md` or organization link
  - issue templates
- [ ] Labels are created.
- [ ] First milestone or bootstrap issues are opened.
- [ ] Maturity/status is stated in README.

## Naming

Repository names should be understandable, not misleading, not impersonating another project, and
not using trademarked names without permission.

## Archival

If a repository becomes inactive or unsafe to maintain, use the archival process in
`docs/PROJECT_LIFECYCLE.md`.
