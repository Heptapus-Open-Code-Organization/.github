# Security Policy

Heptapus Open Code Organization treats security reports seriously, especially for systems software, networking, automation, and infrastructure-like projects.

## Reporting a Vulnerability

If a repository has its own security policy, follow that policy first.

If no repository-specific process exists, contact the organization owner or maintainers privately. Do not publish exploit details in a public issue before maintainers have had a reasonable chance to investigate.

## What To Include

- affected repository and commit/version;
- steps to reproduce;
- expected impact;
- logs, screenshots, or proof-of-concept details when safe to share;
- whether the issue is already public.

## Maintainer Response

Maintainers should acknowledge reports, investigate, document the fix, and credit reporters when appropriate.

## Experimental Projects

Some repositories may be prototypes or research systems. Experimental status does not remove the need for responsible disclosure, but it may affect severity, release timelines, and support expectations.
