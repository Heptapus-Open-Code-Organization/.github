# Contributing to Heptapus Open Code Organization

Thank you for wanting to contribute. This organization is for people who want to build useful open-code systems with seriousness, curiosity, and respect.

## Before You Start

1. Read the repository README.
2. Read the repository license.
3. Check existing issues and work packages.
4. For architectural changes, read existing design notes or ADRs first.
5. Ask before starting a large rewrite.

## Contribution Style

Good contributions usually include:

- a clear problem statement;
- a focused change;
- tests, screenshots, logs, or manual verification where appropriate;
- documentation updates when behavior or architecture changes;
- honest notes about limitations and follow-up work.

## Review Culture

Code review is about improving the work, not winning an argument.

Reviewers should be specific, technical, and respectful. Authors should be open to revision without being expected to accept vague or unsupported criticism.

## Project Maturity

Many projects here may be experimental. Do not claim production readiness unless the repository explicitly says so and the claim is backed by verification.

## Licenses

Each repository defines its own license. By contributing, you agree that your contribution is provided under that repository's license unless a repository-specific policy says otherwise.

## New Project Proposals

A project proposed for the organization should include:

- name and purpose;
- license;
- primary maintainers;
- expected users;
- contribution model;
- current status;
- security or safety considerations.
